Documentation de la nuit de l'info 2023

Introduction

Bienvenue dans la documentation du projet E-Commerce 2023. Notre site web de e-commerce vise à offrir une expérience utilisateur conviviale et efficace, permettant aux utilisateurs d'explorer une variété d'arbres, de les ajouter à leur panier, de s'inscrire, de se connecter, et de valider leurs commandes.

Ce projet a été développé dans le cadre de la Nuit de l'Info 2023, avec pour objectif de sensibiliser les utilisateurs aux enjeux environnementaux tout en proposant une plateforme pratique pour l'achat d'arbres en ligne. Découvrez ci-dessous les principales fonctionnalités et l'architecture MVC qui sous-tend ce projet.

Fonctionnalités

Notre site de e-commerce offre plusieurs fonctionnalités pour une expérience utilisateur complète :

  • Ajout d'arbres au panier : Les utilisateurs peuvent parcourir notre catalogue d'arbres, sélectionner ceux qui les intéressent et les ajouter à leur panier pour une commande ultérieure.
  • Inscription et Connexion : Pour bénéficier de fonctionnalités personnalisées telles que le suivi des commandes, les utilisateurs peuvent s'inscrire, se connecter et gérer leur profil.
  • Validation des Commandes : Une fois connectés, les utilisateurs peuvent valider leurs commandes, fournir les détails d'expédition et effectuer le paiement en ligne de manière sécurisée.

Architecture MVC

Le projet E-Commerce 2023 est construit sur l'architecture Modèle-Vue-Contrôleur (MVC), favorisant une structure logique et modulaire pour le développement web. Voici comment chaque composant est utilisé :

  • Modèle : Le modèle représente la logique métier du projet, notamment la gestion des données. Dans notre cas, le modèle gère les informations sur les arbres, les utilisateurs et les commandes. Les interactions avec la base de données sont gérées dans cette couche.
  • Vue : La vue est responsable de l'affichage des données au client. Elle présente les informations sur les arbres, les formulaires d'inscription, de connexion et de validation des commandes. Elle communique avec le contrôleur pour obtenir les données nécessaires à l'affichage.
  • Contrôleur : Le contrôleur agit comme un intermédiaire entre le modèle et la vue. Il traite les requêtes utilisateur, interroge le modèle pour obtenir des données, puis passe ces données à la vue pour l'affichage. Il gère également les actions utilisateur telles que l'ajout au panier, l'inscription, la connexion et la validation des commandes.

Cette séparation des responsabilités facilite la maintenance du code, permet une évolutivité plus aisée et offre une meilleure organisation du projet.
